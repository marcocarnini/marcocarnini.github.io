Imputing the missing values

Maybe the fact that data are missing is itself a relevant information. I create a column equl to 1 if data are missing and 0 otherwise:

train["coordinates_missing"] = (train["longitude"] < 29).astype(int)
test["coordinates_missing"] = (test["longitude"] < 29).astype(int)

For imputing the values, rather than considering a fixed value like the average or the median of the corresponding coordinates, I would take the average or the median over the district or region codes:

print(len(train["region_code"].unique()))
print(len(train["district_code"].unique()))

20

Retraining the model

Selecting again only the numerical variables without missing data:

X = train.select_dtypes(include=["int64", "float64"])
Y = labels.status_group.ravel()

I train a Random Forest with default hyperparameters:

from sklearn.ensemble import RandomForestClassifier
from sklearn.model_selection import cross_val_score


model = RandomForestClassifier(random_state=2020)
rf_scores = cross_val_score(model, X, Y, cv=10, n_jobs=-1)
print(np.mean(rf_scores), "+/-", np.std(rf_scores))

0.7221548821548822 +/- 0.0052541437547830505

The scores represent only a marginal improvement on the previous result. Reading the previous results and comparing with the current model make obvious that the improvement is marginal:
