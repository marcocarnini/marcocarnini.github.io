Installing the software

I currently work under Ubuntu 18.04 LTS (more properly, Ubuntu on Windows). As I want to use hugo, a fast static site generator written in go. As a first step, I need to install go with the terminal:

sudo apt-get install golang

After installing go, instead of installing hugo through apt-get (the installed version 0.4 would be old 0.4) I followed the following guide. The first step is to download the last deb package from command line:

curl -s https://api.github.com/repos/gohugoio/hugo/releases/latest \
 | grep  browser_download_url \
 | grep Linux-64bit.deb \
 | grep -v extended \
 | cut -d '"' -f 4 \
 | wget -i -

Then the package can be installed (at the moment of writing, the version of hugo is 0.68.3):

sudo dpkg -i hugo_0.68.3_Linux-64bit.deb

Installation was successful, and with the command:

hugo env

I obtained the following output:

Creating a new site locally

From now, I am basically following almost verbatim the quick start guide for hugo. At first, I created the folder for a new site and initializaed a git repository:

hugo new site mysite
cd mysite
git init

Now a theme should be chosen from the gallery. For personal taste I chose Mainroad. The next step is to clone the repository of the theme inside the theme folder (as a submodule):

git submodule add https://github.com/Vimux/mainroad themes/mainroad

Then the theme is added to the configuration file:

echo 'theme = "mainroad"' >> config.toml

Now the configuration file looks like this:

baseURL = "http://example.org/"
languageCode = "en-us"
title = "My New Hugo Site"
theme = "mainroad"

Basic Configuration

First, I edited the configuration file of the blog according to the example in the Github page:

baseURL = "http://example.org/"
languageCode = "en-us"
title = "The Blog of a Curious Data Scientist"
theme = "mainroad"
disqusShortname = "" # Enable Disqus comments by entering your Disqus shortname
googleAnalytics = "" # Enable Google Analytics by entering your tracking id

paginate = "5"

[Author]
name = "Marco Carnini"
bio = "Data Scientist in a Belgian Fintech company. Passionate about Machine Learning and Data Science, I started using R. But then I grew up, and I started using only Python."
avatar = "img/avatar.png"

[Params]
  subtitle = "Books, articles and experiments about Machine Learning" # Subtitle of your site. Used in site header
  description = "Marco Carnini's Personal blog about Data Science" # Site description. Used in meta description
  copyright = "Marco Carnini" # Footer copyright holder, otherwise will use site title
  opengraph = true # Enable OpenGraph if true
  twitter_cards = true # Enable Twitter Cards if true
  readmore = true # Show "Read more" button in list if true
  authorbox = true # Show authorbox at bottom of pages if true
  toc = true # Enable Table of Contents
  post_navigation = true # Show post navigation at bottom of pages if true
  post_meta = ["date", "categories", "translations"] # Order of post meta information
  mainSections = ["posts", "blog", "news"] # Specify section pages to show on home page and the "Recent articles" widget
  dateformat = "2006-01-02" # Change the format of dates
  mathjax = true # Enable MathJax
  mathjaxPath = "https://cdnjs.cloudflare.com/ajax/libs/mathjax/2.7.6/MathJax.js" # Specify MathJax path
  mathjaxConfig = "TeX-AMS-MML_HTMLorMML" # Specify MathJax config
  highlightColor = "#e22d30" # Override highlight color

[Params.sidebar]
  home = "right" # Configure layout for home page
  list = "left"  # Configure layout for list pages
  single = false # Configure layout for single pages
  # Enable widgets in given order
  widgets = ["search", "recent", "categories", "taglist", "social", "languages"]

[Params.widgets]
  recent_num = 5 # Set the number of articles in the "Recent articles" widget
  tags_counter = false # Enable counter for each tag in "Tags" widget

[Params.widgets.social]
  # Enable parts of social widget
  twitter = "marcocarnini"
  linkedin = "marcocarnini"
  github = "marcocarnini"
  gitlab = "fa.mc"
  email = "marcocarnini@gmail.com"

Then, I created an image folder img inside the static folder for containing the avatar (avatar.png); but the content of static will be moved in the folder public, the rootf folder for the site. That’s why the path in the configuration file is for img/avatar.png

Creating a first post

Then, I can generate a first post

hugo new posts/setting-up-hugo.md

I used simply more to inspect the first Markdown post:

more content/posts/setting-up-hugo.md

which produced:

---
title: "Setting Up Hugo"
date: 2020-04-04T20:16:20+02:00
draft: true
---

I added the following lines to the file content/posts/setting-up-hugo.md:

---
title: "Setting up the site"
date: 2020-04-04T20:16:20+02:00
description: "How this site was set up"
summary: "How this site was set up with **hugo** and the theme **mainroad**."
thumbnail: "img/placeholder.jpg" # Optional, thumbnail
disable_comments: true # Optional, disable Disqus comments if true
authorbox: true # Optional, enable authorbox for specific post
toc: true # Optional, enable Table of Contents for specific post
mathjax: true # Optional, enable MathJax for specific post
categories:
  - "Miscellanea"
tags:
  - "Hugo"
---
